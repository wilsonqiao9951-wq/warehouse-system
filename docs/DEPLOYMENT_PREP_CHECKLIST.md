# OpenPartsFlow 首次内部试点 — 部署准备清单

本文与 `docs/PILOT_RUNBOOK.md`、`docs/REAL_DEVICE_TEST_PLAN.md`、`docs/APP_BUILD_GUIDE.md` 配合使用。部署前由负责人逐项勾选并保留证据（截图、配置版本号、备份文件名）。

---

## 1. 前端环境变量

| 变量 | 说明 | 试点建议 |
|------|------|----------|
| `NEXT_PUBLIC_API_BASE_URL` | 浏览器可调用的 **FastAPI API 根路径**（须以 `/api` 结尾，与当前前端 `lib/api.ts` 约定一致） | 生产/试点 HTTPS 域名，例如 `https://api.pilot.example.com/api`；勿在构建产物中硬编码内网 IP 到公网环境 |

**注意**

- 该变量在 **`next build` 时打入静态包**，变更 API 地址后需 **重新构建** 前端并重新部署 `out`（或等价托管）。
- 前端与 API **跨域** 时，后端必须配置 **CORS** 允许前端来源。
- 真机验证见 `docs/REAL_DEVICE_TEST_PLAN.md`（局域网 IP、防火墙）。

参考：`frontend/.env.example`。

---

## 2. 后端环境变量

与 `app/core/config.py` 及根目录 `.env.example` 对齐：

| 变量 | 说明 | 试点建议 |
|------|------|----------|
| `APP_NAME` | 应用名称 | `OpenPartsFlow` |
| `APP_ENV` | 环境标识 | `staging` 或 `production`（勿长期用 `development` 跑试点生产库） |
| `APP_DEBUG` | 调试模式 | **`false`** |
| `LOG_LEVEL` | 日志级别 | `INFO`（排障时可临时 `DEBUG`，结束后改回） |
| `DATABASE_URL` | 数据库连接串 | 试点独立库；SQLite 仅适合极小流量验证，**正式试点推荐 PostgreSQL** |
| `RBAC_ENFORCE` | 是否强制 RBAC | 见下文 **§8 RBAC_ENFORCE 上线节奏** |

**安全（补充）**

- 生产勿将 `.env` 提交到仓库；使用密钥管理或 CI 注入。
- 若使用 `X-User-Id` 头模拟登录，须在网关或内网层限制来源，避免公网任意伪造（长期应过渡到正式认证）。

## 4. 文件上传存储方案

当前后端将工单零件相关图片写入服务器本地目录（如 `uploads/work-order-parts/`，见 API 路由实现）。

- [ ] **路径**：部署用户对该目录有读写权限；容器场景需 **volume 挂载**。
- [ ] **持久化**：实例重启、滚动发布不丢文件；多副本时需 **共享文件系统** 或 **对象存储（S3 兼容等）** — 若未来改为对象存储，属架构变更，需单独变更窗口。
- [ ] **备份**：上传目录纳入备份或同步策略（与数据库备份频率协调）。
- [ ] **容量与清理**：试点前评估单张大小与数量；制定保留策略（合规要求下可能需更长保留）。

## 8. RBAC_ENFORCE 上线节奏

`RBAC_ENFORCE`（`.env` 中 `true`/`false`）控制是否严格按角色限制 API。

| 阶段 | 设置 | 目的 |
|------|------|------|
| **迁移验证期**（可选 0.5～1 天） | `false` | 快速打通链路、修配置；**仅限受控网络** |
| **内部试点主阶段** | **`true`** | 与真实权限一致，暴露越权与错误 `X-User-Id` 问题 |
| **若大量 403 阻塞业务** | 临时 `false` + 记录事故单 | 仅限短窗口排障，并尽快恢复 `true` |

- [ ] 试点 **Day 1 开始前** 已确认主阶段为 `true` 的时间点。
- [ ] 全员知晓：**角色与 `X-User-Id` 必须与后端用户一致**（见培训文档）。

## 9. Windows / 本地测试 与 生产 的差异

| 维度 | 本地 / Windows 开发机 | 生产 / 试点 HTTPS |
|------|------------------------|---------------------|
| API 地址 | 常為 `http://127.0.0.1:8000/api` | HTTPS 域名 |
| 前端访问 | localhost 或局域网 IP | 域名 + CDN |
| PWA / SW | localhost 例外；手机局域网 HTTP 功能可能受限 | 完整安装与 SW 行为以 HTTPS 为准 |
| 防火墙 | 需手动开端口供手机测 | 由云安全组 / WAF 管理 |
| 数据 | 本地 SQLite 或临时库 | 独立试点库 + 备份 |
| RBAC | 易误用 `false`「图方便」 | 强制 `true` 贴近真实 |

- [ ] 已在 **类生产 HTTPS** 环境至少跑通一次端到端（登录模拟、工单、上传、报表）。

## 10. 回滚到 AppSheet 的计划

与 `docs/PILOT_RUNBOOK.md` §6 一致，部署侧补充：

1. **触发条件**：严重库存差异、RBAC 越权、大面积无法完工、数据损坏等（见 Runbook）。
2. **动作**：公告暂停试点 → **AppSheet 恢复为主记录** → OpenPartsFlow **只读**调查。
3. **数据保全**：导出 WO、库存、零件、审计日志、异常用量（Runbook 所列端点）；备份文件校验哈希或签名。
4. **责任**：指定 **单一决策人** 宣布回滚与恢复 AppSheet 流程的时间点。
5. **再试点**：根因修复、变更记录后再排期小规模试点。

- [ ] 回滚联系人、导出操作步骤已发给经理与仓库主管纸质或 PDF 一页纸。
