# OpenPartsFlow 员工培训指南（内部试点版）

适用对象：试点技师、经理、仓库人员。  
说明：当前 Web 应用通过浏览器使用（含手机 PWA）。登录方式为页面顶部 **角色** 与 **`X-User-Id`**（你的用户数字 ID）；正式 SSO 上线前请妥善保管自己的 ID，勿使用他人 ID。

---

## 1. 技师（Technician）培训

### 1.1 每天开始

1. 用手机或电脑打开公司提供的 **OpenPartsFlow 网址**（HTTPS）。  
2. 顶部 **角色** 选 **technician**。  
3. 在 **X-User-Id** 输入你的 **数字工号/用户 ID**（由主管下发），输入后会保存在本机浏览器。  
4. 底部导航：**Today**（今日概览）、**Map**、**My Jobs**、**My Van**、**Profile**。

### 1.2 如何开始工单（Start job）

1. 打开 **My Jobs** 或 **Today**。  
2. 在卡片上点击 **Start**（或在 **工单详情** 页点击 **Start job**）。  
3. 若提示错误：看红色说明；常见为网络断开、`X-User-Id` 错误、或工单已锁定。

### 1.3 如何记录用料（Use parts）

1. **Parts Usage** 页面通常由 **仓库/管理员** 代录；若你被授予该菜单权限：  
   - 选择 **工单、零件、仓库（一般为你的 Van 仓）、数量**。  
   - 可选 **拍照** 或 **相册** 选图，确认 **预览** 后再提交。  
2. 提交成功后系统会从库存扣减；失败时不要重复狂点，截图联系支持。

### 1.4 如何上传 QC 照片

1. 打开 **工单详情**（从 My Jobs 点 **Open**）。  
2. 在 **QC pictures** 区域：  
   - 点 **Take QC photo**（现场拍照）或 **Choose from library**；  
   - 或粘贴已有图片 **URL**；  
3. 看到预览后，点 **Add QC picture**；上传过程中请等待 **Working…** 结束。  
4. 成功后列表会出现缩略图；打不开图片时把问题反馈给支持（多为网络或权限）。

### 1.5 如何完工（Complete job）

1. 确认现场工作已完成、用料与 QC 已按公司要求录入。  
2. 在 **工单详情** 或列表卡片点 **Complete** / **Complete job**。  
3. 完工后工单会 **锁定**，无法再改状态或补录——务必在点 Complete 前自检。

### 1.6 如何查看车上库存

1. 打开 **My Van**（My Van Inventory）。  
2. 查看数量；若显示异常低，先不要自行改数据，通知经理或仓库。

---

## 2. 经理（Manager）培训

1. 角色选 **manager**，填写你的 **`X-User-Id`**。  
2. **Dashboard**：总览 KPI。  
3. **Work Orders**：筛选、分页、分配技师、打开详情；窄屏为卡片、宽屏为表格。  
4. **Reports**：关注 **异常用量** 高亮行。  
5. **Pilot Checklist**：试点健康度一览。  
6. **Inventory**：关注 **低库存告警** 区块。  
7. 每日结束：按 Runbook 做 **导出备份**（如有要求）。

---

## 3. 仓库（Warehouse）培训

1. 角色选 **warehouse**，填写 **`X-User-Id`**。  
2. **Inventory**：查看仓库与 Van 汇总、低库存。  
3. **Parts Usage**：为工单录入用料（选工单、零件、仓库、数量；可附照片）。  
4. 调拨/事务类流程以公司 SOP 为准（与系统菜单一致处按界面操作）。  
5. 任何批量导入后，与经理做一次 **小样本对账**。

---

## 4. 绝对不要做的事（What not to do）

- 不要用别人的 **`X-User-Id`** 登录，以免造成错误记录与权限纠纷。  
- 不要在 **未完工** 前随意点 **Complete**；锁定后无法自行撤销。  
- 不要在 **离线** 时反复提交写入操作——系统不会假装保存成功；请恢复网络后重试。  
- 不要向公司外部人员泄露试点 URL、导出文件、或管理员账号。  
- 不要依赖 **HTTP 局域网地址** 作为最终正式流程（正式试点以 **HTTPS** 为准）。

---

## 5. 出问题找谁（Who to contact）

| 问题类型 | 第一联系人 | 说明 |
|----------|------------|------|
| 无法打开网页、证书错误 | IT / 部署负责人 | 域名、HTTPS |
| 403、权限、角色错误 | 技术负责人 + 你的主管 | `RBAC_ENFORCE`、用户角色 |
| 库存数量不对 | 仓库主管 + 技术负责人 | 对账、交易记录 |
| 手机拍照上传失败 | 技术负责人 | API 地址、CORS、上传目录 |
| 业务规则不清 | 运营经理 | 与 AppSheet 流程对齐 |

**上报时请附带**：时间、你的角色、用户 ID、工单号、截图、是否离线。
